Most NFC tags are identified by a 4-, 7-, or 10-byte unique serial number (UID). The tag will also return additional information during the detection procedure that can be used to identify what type of tag it is. However, there are collisions between certain types of tags, in which case the Tappy will report either a generic type or attempt to heuristically determine the tag technology, depending on configuration. If the Tappy reports that a tag is a 'generic' type, this means that the heuristics were unable to determine the specific tag technology to a sufficient level of confidence. Additionally, for security reasons, some advanced tags can be configured to report a dynamic UID, in which case the UID detected will change with every scan.
